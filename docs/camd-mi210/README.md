# CAMD MI210 Communication Findings

These notes record the communication-health investigation that motivated the
standalone benchmark.

They are intentionally cluster-specific. The benchmark itself is in the repository
root and can be run without these historical notes.

Useful entry points:

- `COMM_CHECK_LOOP.md`: benchmark motivation, methodology, and execution loop.
- `global_node_summary.md`: node-level classification table and category legend.
- `summary_2nodes.md`, `summary_4nodes.md`, `summary_8nodes.md`,
  `summary_16nodes.md`: scale-specific observations.
- `2nodes_faulty.md`: focused notes on faulty two-node cases.
- `SUBMITTED_JOBS.md`: job tracking from the investigation.
